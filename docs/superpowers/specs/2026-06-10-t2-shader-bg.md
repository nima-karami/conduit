# T2 — Real WebGL shader background

## Problem
"Shader" was requested; only CSS + a 2D-canvas Flow were built. No actual GLSL.

## Fix
Add a `shader` background mode: a WebGL fullscreen fragment shader rendering a
flowing domain-warped plasma, coloured from the theme (`--accent/--blue/--violet`),
with intensity controlling alpha. Perf-aware (rAF, pause when hidden), honours
reduce-motion, handles resize + context loss, and **falls back to the Flow canvas
if WebGL is unavailable**.

## Implementation
- `webview/components/ShaderBg.tsx`: sets up a WebGL1 context + fullscreen-quad
  program; fragment shader = layered sin plasma; uniforms `u_res, u_time, u_c1..c3,
  u_alpha`. Colours parsed from CSS vars; alpha from intensity. Returns `null` (so
  the caller can fall back) when `getContext('webgl')` fails.
- `settings`: add `'shader'` to Background + BACKGROUNDS list; SettingsModal BG_OPTS
  adds "Shader".
- `AnimatedBg`: for `background==='shader'` render `<ShaderBg>` (reduceMotion → null);
  if ShaderBg signals no WebGL, render the Flow canvas instead.

## Acceptance criteria
1. Selecting Shader shows an animated WebGL plasma coloured by the theme.
2. Intensity changes its strength; theme change recolours it.
3. reduceMotion disables it; animation pauses when hidden.
4. Graceful fallback to Flow when WebGL is unavailable.
5. typecheck + build + tests green.
