# LaTeX math rendering in the Markdown viewer (KaTeX)

## Problem frame

**Job:** A user reading a technical doc with LaTeX math wants the equations typeset,
not shown as raw `$‚Ä¶$` / `$$‚Ä¶$$` source. Conduit's viewer currently passes math
delimiters through untouched, so `$E=mc^2$` renders literally.

**Actor:** anyone viewing a `.md` with inline or block math (specs, papers, notes).

**Success:** inline `$x$` and block `$$‚Ä¶$$` math render as typeset KaTeX; non-math
docs are byte-for-byte unaffected; the rest of the pipeline (gfm, highlight, mermaid,
links, reveal) keeps working.

**Non-goals:** MathML fallback; a math editor; supporting non-`$` delimiters beyond
what remark-math handles by default; server-side rendering.

## Behavior & states

- Add `remark-math` to `remarkPlugins` and `rehype-katex` to `rehypePlugins` in
  `MarkdownViewer`'s `<ReactMarkdown>`. Order matters: `remark-math` parses math
  nodes before remark‚Üírehype; `rehype-katex` must run in the rehype phase.
- KaTeX stylesheet is imported so fractions/roots/scripts lay out correctly (KaTeX
  is unstyled without its CSS). Import `katex/dist/katex.min.css` once in the viewer
  module (same pattern as the existing `highlight.js/styles/github-dark.css` import).
- A malformed expression renders KaTeX's own error markup (red) inline rather than
  throwing ‚Äî `rehype-katex` is configured with `throwOnError:false` (default) so one
  bad formula never blanks the whole doc.

## Data / interface contract

No new app module. Plugin wiring + CSS import only. New runtime deps:
`remark-math`, `rehype-katex`, and their `katex` peer ‚Äî all added to
`package.json` dependencies (so `fallow:check`'s unlisted-deps gate stays green).

Interaction with existing rehype plugins: current order is
`[rehypeSourceLine, rehypeHighlight]`. KaTeX output is a code-free subtree, so
`rehype-katex` is independent of `rehypeHighlight`; place it so it does not strip the
`data-source-line` attributes `rehypeSourceLine` stamps (append after, don't replace).

## Edge cases & failure modes

- **Currency/`$` that isn't math** (e.g. "it cost $5 and $10") ‚Äî remark-math only
  treats balanced `$‚Ä¶$` without surrounding digits-adjacency per its rules; accept
  remark-math's standard behavior (don't hand-roll delimiter logic). Document as a
  known remark-math semantic, not a Conduit bug.
- **Malformed LaTeX** ‚Üí inline KaTeX error (red), doc still renders. (no throw)
- **Block math inside lists/blockquotes** ‚Üí handled by remark-math node nesting.
- **Mermaid/code blocks** must be untouched ‚Äî math only applies to `$` text nodes,
  not fenced code. Verify a doc with both renders both.
- **CSP / fonts:** KaTeX ships its own woff2 fonts referenced by its CSS. The esbuild
  webview build must emit/serve those fonts (loader for woff2) OR KaTeX falls back to
  the bundled CSS's base64-free references ‚Äî confirm glyphs render in the harness; if
  fonts 404, add a `.woff2`/`.woff` loader entry to esbuild's web config.

## Defaults vs. settings

No setting. Math always renders ‚Äî there is no reasonable "show raw `$$`" preference
for a doc reader. Rationale: a toggle would be divergent surface nobody asks for; the
source view (existing "View source") already shows raw delimiters when wanted.

## Scope slicing

- **MVP:** inline + block KaTeX via remark-math/rehype-katex + CSS; fonts resolve in
  the harness; non-math docs unchanged.
- **Out of scope:** equation numbering, `\tag`, macros config, copy-as-LaTeX.

## Acceptance criteria

- AC1: A doc with inline `$a^2+b^2=c^2$` and a block `$$\int_0^1 x^2\,dx = \tfrac13$$`
  renders typeset equations (no literal `$` delimiters) in the preview harness.
  (playwright screenshot + assert `.katex` elements exist)
- AC2: A doc with a `$`-as-currency line and a mermaid block both still render
  correctly (no regression to mermaid/code). (playwright)
- AC3: KaTeX glyphs lay out correctly ‚Äî fonts load (no 404 for katex fonts in the
  console). (playwright console check)
- AC4: `npm run verify` stays green (deps listed, build emits fonts, types pass).
- AC5: A doc with NO math is unchanged vs. before (headings/tables/code identical).
  (playwright snapshot diff or visual)

## Decisions Needed

none ‚Äî standard remark-math/rehype-katex defaults chosen; no delimiter customization.
