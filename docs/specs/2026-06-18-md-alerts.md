# GitHub-style alerts / admonitions

## Problem frame

**Job:** Markdown authors (and agents writing docs) use GitHub's alert syntax to
call out notes/warnings:

```
> [!NOTE]
> Useful information a reader should know.
```

Conduit currently renders these as a plain blockquote with a literal `[!NOTE]` line
— the callout intent is lost.

**Actor:** anyone viewing a `.md` using `> [!TYPE]` alerts (very common in READMEs,
agent-authored docs).

**Success:** the 5 GitHub alert types — `NOTE`, `TIP`, `IMPORTANT`, `WARNING`,
`CAUTION` — render as colored callouts with a type icon + label, matching GitHub's
visual language and themed to Conduit's palette. Plain blockquotes are untouched.

**Non-goals:** custom/arbitrary alert types; collapsible alerts; nested-alert
styling beyond inheritance.

## Behavior & states

- A blockquote whose **first line** is exactly `[!TYPE]` (case-insensitive, one of
  the 5) becomes a callout: container `div.markdown-alert.markdown-alert-<type>`
  with a `div.markdown-alert-title` (icon + capitalized label) followed by the
  remaining blockquote content. The `[!TYPE]` marker line is removed from the body.
- An unrecognized marker (`[!FOO]`) or a blockquote without a leading marker →
  rendered as a normal blockquote (no change). Belt: matching is exact on the first
  text token.
- Each type has a distinct accent color + icon (info/bulb/report/warning/stop),
  applied via CSS using the app's semantic tokens (`--blue`, `--green`, `--accent`,
  `--amber`, `--red`) so it themes with the app.

## Data / interface contract

New `webview/md-alerts.ts` — a **remark** plugin `remarkAlerts()` (pure mdast
transform, unit-testable without DOM):

- Visits `blockquote` nodes; inspects the first child `paragraph`'s first `text`
  node for `^\[!(note|tip|important|warning|caution)\]` (case-insensitive), allowing
  an immediate newline/soft-break after the marker.
- On match: strips the marker (and following break) from the paragraph; if the
  paragraph becomes empty, drops it. Sets the blockquote's
  `data.hName='div'` + `data.hProperties.className=['markdown-alert',
  'markdown-alert-'+type]`. Prepends a title node
  (`paragraph` → `data.hName='div'`, class `markdown-alert-title`, text = Label).
- Exposes `ALERT_TYPES` (the 5) and a `matchAlertType(text): type|null` helper for
  the unit test.

Wire into `MarkdownViewer`'s `remarkPlugins` (after `remarkGfm`, before/after
`remarkMath` — independent). Icons are drawn via CSS `::before` on
`.markdown-alert-title` using inline-SVG `mask-image` per type (CSP-safe, no JS,
themeable via `background-color`).

## Edge cases & failure modes

- **No marker** → normal blockquote (default path). (regression-critical)
- **Unknown type** → normal blockquote.
- **Marker but empty body** (`> [!NOTE]` alone) → callout with title only, no body.
- **Marker not on first line** (text before it) → not an alert.
- **Lowercase/mixed case** (`[!warning]`) → matched, label normalized to title-case.
- **Interaction with `data-source-line`** — `rehypeSourceLine` still tags the
  resulting `div`; reveal/scroll keeps working.

## Defaults vs. settings

No setting — alerts always render (matches GitHub/VS Code/Obsidian). Rationale:
universal, non-divergent; the marker is opt-in by the author already.

## Accessibility / i18n

- The title conveys type by **icon + text label** (not color alone) — color-blind
  safe. Container gets `role="note"` via hProperties for AT.
- Labels are the canonical English type names (NOTE/TIP/…), matching the syntax
  tokens themselves; not user content, so not localized (consistent with the rest of
  the app's English-only UI).

## Scope slicing

- **MVP:** 5 types, icon+label+color, dep-free remark plugin, themed CSS.
- **Out of scope:** collapsible/foldable alerts, custom titles
  (`> [!NOTE] Custom title`).

## Acceptance criteria

- AC1: A doc with all 5 alert types renders 5 distinct colored callouts each with an
  icon + label; the `[!TYPE]` marker text is gone. (playwright screenshot + assert
  `.markdown-alert` count = 5)
- AC2: A plain blockquote (no marker) still renders as a normal blockquote (no
  `.markdown-alert` class). (playwright + unit)
- AC3: `matchAlertType` returns the type for valid markers (any case) and null for
  unknown/absent. (unit)
- AC4: An empty-body alert (`> [!NOTE]`) renders a title-only callout without
  crashing. (unit + playwright)
- AC5: `npm run verify` green; existing blockquote rendering unaffected.

## Decisions Needed

none — conventional GitHub semantics + dep-free plugin (matches the repo's existing
custom-plugin approach in md-reveal.ts).
